01

Princípio central

O Sistema de Produção é uma infraestrutura leve de gestão do projeto A Floresta que Fala. Ele integra site, Apps Script, Google Sheets, Google Drive e, futuramente, Google Calendar, mantendo simplicidade operacional para a equipe.

02

Princípio da Evolução Incremental

O sistema deverá ser desenvolvido em pequenas etapas operacionais.

Cada nova funcionalidade deve ser útil por si só, testável e documentada antes da próxima implementação.

Devem ser evitadas grandes refatorações ou funcionalidades extensas que aumentem a complexidade sem gerar benefício imediato para a equipe.

Aplicação prática

  • Implementar uma função de cada vez.
  • Testar antes de avançar.
  • Documentar cada mudança relevante.
  • Registrar marcos temporais quando houver avanço significativo.
  • Evitar duplicação de código.
  • Preservar compatibilidade com o sistema já existente.
  • Priorizar clareza para usuários não técnicos.
Antes de crescer, o sistema precisa continuar funcionando.

07

Regras obrigatórias

  • Deve existir apenas um doGet().
  • Deve existir apenas um doPost().
  • A planilha é a fonte oficial dos dados.
  • O site apenas envia e consulta dados.
  • Não apagar histórico sem autorização.
  • Auditorias devem ser acumulativas.
  • Formulários simples antes de painéis complexos.
  • Não implementar login complexo nesta fase.
  • Não criar dashboards antes de existir histórico suficiente.
  • Toda nova função deve indicar qual aba usa.
  • Toda nova função deve indicar se grava, consulta ou audita dados.
08

Procedimento antes de nova implementação

Antes de criar algo novo, verificar:

  1. Já existe função parecida?
  2. Já existe aba para esse tipo de dado?
  3. A função deve ficar em qual módulo?
  4. Ela precisa gravar dados?
  5. Ela precisa retornar JSON?
  6. Ela altera histórico?
  7. Ela será usada pelo site?
  8. Ela precisa aparecer no dashboard?

Aviso para futuras IAs / colaboradores

Antes de implementar qualquer nova função neste sistema, leia este manual.

Não duplique funções existentes.

Não crie novas abas sem necessidade.

Não altere nomes oficiais sem atualizar este documento.

Preserve simplicidade, histórico e rastreabilidade.
